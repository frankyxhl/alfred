## What

Promote the useful parts of `BAB-1504-SOP-GitHub-Codex-PR-Review-Loop.md` into a PKG-layer COR SOP so every Alfred project can use the workflow. The promoted SOP is `COR-1615: GitHub App PR Review Bot Loop`.

The new COR SOP generalizes the Babs-specific Codex wording into a GitHub App review bot loop: trigger once per PR head, poll without spam, interpret reactions conservatively, match review results to the current `headRefOid`, fetch thread-aware state when staleness matters, and hand actionable findings to COR-1612.

## Why

BAB-1504 captured a real process gap: agents can mistake an acknowledgement reaction for review completion, re-fix stale inline comments, or treat a review for an older commit as clearing the latest PR head. COR-1612 already governs response to fetched findings, but it does not provide a concise entry SOP for requesting and polling GitHub App bot review passes. Promoting the pattern to COR makes it reusable outside Babs.
