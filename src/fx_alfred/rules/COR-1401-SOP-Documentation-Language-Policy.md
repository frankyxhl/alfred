# SOP-1401: Documentation Language Policy

**Applies to:** All projects using the COR document system
**Last updated:** 2026-03-12
**Last reviewed:** 2026-03-12
**Status:** Active

---

## What Is It?

A policy that standardizes the language used in all project documentation. This ensures consistency, searchability, and readability across all documents regardless of the team's spoken language.

---

## Policy

1. **All documents must be written in English.** This includes SOPs, INC records, PLN documents, CHG requests, ADR records, REF documents, and any future document types.
2. **Verbal and chat discussions may happen in any language.** The working language between team members is not restricted.
3. **When recording a decision or outcome from a non-English discussion**, translate the key points into English for the written document.

---

## Scope

This policy applies to:

- All documents across all layers (PKG, USR, PRJ) in every project (ALF, BLA, CLR, and any future projects)
- All document types in the numbering system (SOP, INC, PLN, CHG, REF, ADR)
- Code comments and commit messages (recommended but not enforced)

---

## Exceptions

- Inline examples or quotes that only make sense in the original language may be kept as-is, with an English translation in parentheses if needed
- File names are always in English (already enforced by kebab-case naming convention)
